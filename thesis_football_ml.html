Abstract
ABSTRACT · Page iii
The incorporation of machine learning algorithms in various domains has revolutionised prediction systems, achieving unmatched accuracy. By employing feature engineering and exploratory data analysis, a tailored set of features was developed to reveal the crucial factors influencing football match results. The study examines the last 23 seasons of the English Premier League, analysing and combining actual match data with bookmakers' and betting data. Six models were executed with Grid Search and Randomised Search. Logistic Regression topped all models with 96% accuracy — surpassing prior state-of-the-art results achieved on far narrower datasets. Novel features — particularly streak and form metrics — proved to have crucial impact on predictive performance.

Chapter 1 — Introduction
1.1 Introduction

Data mining and predictive analysis have become essential tools in multiple fields, enabling extraction of important insights from data for research and practical use. These techniques range widely from detection of financial crimes to analysis of sports dynamics. Football (soccer) is widely regarded as the most esteemed sport worldwide. The English Premier League is a prime example of sporting brilliance, attracting a large number of viewers and exerting substantial economic impact.

Forecasting the outcomes of football matches poses a daunting task due to the intrinsically competitive and capricious nature of the sport. Prior investigations in football match prediction are broadly classified into two streams: outcome-oriented studies (predicting Win/Draw/Loss) and goal-oriented studies (predicting goals scored). The combination of data mining and predictive analysis holds great potential to improve prediction accuracy, especially in the sports betting sector.

Football and computer science have numerous intersections in the modern environment, transforming player analysis, tactical plans, scouting, and fan engagement. Wearable sensors and ML algorithms provide coaches with detailed performance metrics and opponent tendencies.

1.4 Thesis Objectives

To accurately forecast football matches using Premier League data, incorporating football statistics of the last 23 years, bookmakers' data and betting odds. The models will be refined to enhance accuracy and incorporate new features through feature engineering. The study compares various methods and optimises hyperparameters using grid and randomised search.

THEORY

Feature engineering optimises model performance, decreases complexity, and boosts interpretability. Feature reduction techniques include PCA and t-SNE. The Correlation Matrix quantifies relationships between variables; the correlation coefficient rxy ranges from −1 to +1. Multicollinearity (two features highly correlated) is removed.

Standardisation formula:
z = (x − μ) / σ
where z = standardised value, μ = mean of feature, σ = standard deviation

Correlation coefficient r_xy quantifies linear association between two variables — ranges [−1, +1]
2.3.5 Machine Learning Models — Technical Overview
LR
Logistic Regression
Applies sigmoid σ(z)=1/(1+e⁻ᶻ) to a linear combination of features. Despite its simplicity, it achieved the highest accuracy — a testament to the quality of engineered features and linear separability of the processed data. Cost function: binary cross-entropy. Optimised via gradient descent.
SVM
Support Vector Machine
Seeks hyperplane maximising margin between classes: wᵀx + b = 0. Primal objective: minimise ½‖w‖² subject to y⁽ⁱ⁾(Wᵀx⁽ⁱ⁾+b) ≥ 1. Grid Search outperformed Randomised Search for SVM (90%) — second-highest individual result.
RF
Random Forest
Ensemble of decision trees on bootstrap samples; majority vote for classification. Despite popularity in sports prediction literature, it underperformed here (77%) — suggesting the EPL data's structure favours linear over tree-based decomposition.
ADA
AdaBoost
Adaptive Boosting — iteratively trains weak classifiers (decision stumps), assigning higher weight to misclassified samples. Achieved 82% accuracy. Randomised Search underperformed Grid Search for this model, suggesting its hyperparameter space is more structured.
XGB
XGBoost
Gradient Boosted Trees with L1 (reg_alpha) and L2 (reg_lambda) regularisation. Builds trees sequentially — each corrects residuals of prior. Achieved 84% with Randomised Search. Performance gap with LR confirms feature engineering reduced complexity advantage of tree-based methods.
ENS
Ensemble + SMOTE
Hard Voting combining AdaBoost, XGBoost, SVC, and LR. SMOTE variant achieved 91% — second-best overall. Synthetic Minority Over-sampling creates artificial instances for minority class. Demonstrates that class balance is a significant factor in this dataset.
2.3.6 Hyperparameter Tuning — Grid vs Randomised
Grid Search
Exhaustively evaluates every combination within a specified parameter grid. Computationally expensive but guaranteed to find the best combination within the defined space. Steps: (1) Define grid → (2) Train per combination → (3) Cross-validate → (4) Select best. Better results for SVM, Random Forest, and AdaBoost in this study.
Randomised Search
Samples random combinations from parameter distributions. More efficient for large search spaces; often finds near-optimal configurations. Steps additionally require: specifying number of iterations and defining probability distributions. Better results for Logistic Regression and XGBoost — when optimal hyperparameters aren't predictable from domain knowledge, random sampling finds superior configurations more efficiently.

3.3.3 Dataset Preprocessing — All 16 Steps
Step 01
Goals Scored and Conceded at End of Matchweek
Track GSt,m and GCt,m cumulatively — goals scored/conceded by team t up to matchweek m.
Step 02
Define get_points(result) Function
Points = 3 (win), 1 (draw), 0 (loss). Facilitates calculation of team standings.
Step 03
Calculate Cumulative Points
CPt,m = Σ points earned by team t up to matchweek m.
Step 04
Get Match Results
get_matchres() assigns 'W', 'D', or 'L' to each match based on full-time result.
Step 05
Get Team Form
Recent form based on last few matches. Represented as sequence e.g. 'WWDLW' for last five.
Step 06
Get Matchweek
Assign matchweek numbers (MW) to each match for temporal tracking.
Step 07
Get Team Form Points
FPt,m = Σ points earned in last n matches up to matchweek m.
Step 08
Identify Win or Loss Streaks
Detect consecutive wins or losses. Analyses momentum and performance trends.
Step 09
Get Goal Difference
ΔGDt,m = GSt,m − GCt,m. Reflects offensive and defensive capabilities.
Step 10
Get Difference in Points
ΔPtt1,t2,m = CPt1,m − CPt2,m. Compares relative performance between teams.
Step 11
Scale DiffPts, DiffFormPts, HTGD, ATGD by Matchweek
scale_by_MW(x) = x / MW. Normalises metrics across different points in season for fair comparison.
Step 12
Build Correlation Matrix (Fig. 3.15)
Calculate correlation coefficients for all feature pairs. Identifies redundant features for removal.
Step 13
Remove Highly Correlated Columns
Eliminate columns based on redundancy, irrelevance, or high correlation. Prevents multicollinearity.
Step 14
Separate Feature Set (X) and Target Variable (y)
X_all = all columns except 'FTR'. y_all = 'FTR' column (Full-Time Result).
Step 15
Standardise Selected Columns
z = (x−μ)/σ applied to 'HTGD', 'ATGD', 'HTP', 'ATP'. Mean = 0, SD = 1.
Step 16
Convert Categorical Variables into Dummy Variables
One-hot encoding: xij = 1 if i-th observation belongs to category j; 0 otherwise.

4.2 Evaluation Metrics
True Positive (TP)

Model correctly predicts positive when actual outcome is positive. (Test correctly detects disease when present.)

True Negative (TN)

Model correctly predicts negative when actual outcome is negative. (Test correctly rules out disease when absent.)

False Positive (FP)

Model incorrectly predicts positive when actual is negative. (False alarm — disease flagged when not present.)

False Negative (FN)

Model incorrectly predicts negative when actual is positive. (Miss — disease not detected when actually present.)

Accuracy = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1-Score = 2 × (Precision × Recall) / (Precision + Recall)

Macro Avg Precision = (Precision_A + Precision_B + Precision_C) / 3
Weighted Avg = Σ(Precision_class × Support_class) / Total_support

Chapter 5 — Conclusion & Future Works
A 96% accuracy that reshapes the field.
Previous research in football match prediction achieved a ceiling of approximately 85% accuracy using Random Forest on limited seasonal data. This thesis pushed beyond that boundary by combining three key innovations: a much larger dataset (23 EPL seasons), the integration of bookmakers' odds data alongside traditional match statistics, and carefully engineered features — particularly streak and form metrics — that capture the temporal momentum of teams across a season.

The finding that Logistic Regression outperformed far more complex models is not a failure of complexity — it is a demonstration of what proper feature engineering can accomplish. When the input data is well-structured and features are meaningful, a linear model's interpretability becomes an advantage rather than a limitation.

The study also demonstrates the real cost of narrow datasets. Prior work based on two seasons or cup events cannot generalise conclusions about EPL football dynamics. Twenty-three seasons of data reveals patterns that are simply invisible in shorter windows.

The model does not yet generalise well to other leagues — highlighting the league-specific nature of learned patterns and the complexity of football prediction beyond the EPL.

Bookmakers' data substantially improved predictive accuracy when fused with match stats

Streak & form — engineered features proved crucial to model performance

Linear models can outperform ensembles with quality feature engineering

5.2 Future Research Directions
Direction 01
Psychological factors
Incorporate sentiment analysis and player psychology indicators. Real-time tracking data and social media signals to capture team morale and public sentiment ahead of matches.
Direction 02
Multi-league expansion
Expand dataset to cover top European leagues and international tournaments. The current model's cross-league limitation is explicitly acknowledged — a generalised model remains an open problem.
Direction 03
Player-level granularity
Incorporate individual player performance metrics, fitness tracking, and injury probability estimates. Current models operate at team level — player-level features could significantly improve accuracy where lineup changes are decisive.
Direction 04
Managerial intelligence
Integrate managerial records, tactical preferences, and in-game decision-making patterns. A manager change mid-season is known to significantly affect team performance — this signal is currently absent from the model.
Direction 05
Real-time integration
Build a live prediction system updating in-game using real-time event data. The current model predicts pre-match — real-time integration could produce a dynamic probability model adjusting as the match unfolds.
Direction 06
Advanced architectures
Explore LSTMs for sequential match-form modelling, Transformers for player-interaction graphs. The 96% accuracy baseline set here provides a strong benchmark for these explorations.
